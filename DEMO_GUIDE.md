## Demo Q&A Prep

### Q: "How long does onboarding take?"
**A**: "Design-time (agents generate code): 5-10 minutes. Runtime (first data processing): 30-45 minutes. After that, it's automated daily."

### Q: "What if data quality fails?"
**A**: "Pipeline blocks at quality gate. Silver requires ≥ 80%, Gold ≥ 95%. Agent logs show which checks failed. Fix data and re-run."

### Q: "Can it handle real-time data?"
**A**: "Current implementation: batch (daily). Architecture supports real-time via EventBridge + Lambda, but not yet implemented."

### Q: "What about data governance?"
**A**: "Full lineage tracked. MCP logs show every operation. Lake Formation for access control. SageMaker Catalog for business metadata. Audit-ready."

### Q: "How does it scale?"
**A**: "Currently 4 workloads. MWAA supports 20+ with current config. Add workers for more. Architecture is workload-independent."

### Q: "What's the ROI?"
**A**: "Traditional: 3 weeks × $200/hr × 80 hrs = $16,000 per dataset. Our platform: 3 hours × $200/hr = $600. Break-even after ~1 dataset."

### Q: "Why MCP vs direct boto3?"
**A**: "Standardization. Every AWS call goes through MCP → logged, timed, tracked. Audit trail for compliance. Easy to swap providers (multi-cloud)."

### Q: "Can I add my own MCP servers?"
**A**: "Yes! We built 4 custom servers (SageMaker Catalog, local-filesystem, EventBridge, Lake Formation) using FastMCP. Takes ~30 minutes."
