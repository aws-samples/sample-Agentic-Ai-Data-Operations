┌─ THE 7 STEPS ───────────────────────────────────────────────────────────────┐
│                                                                             │
│  1. Read semantic.yaml              ← YAML configuration                   │
│  2. Fetch Glue Catalog metadata     ← Technical schemas                    │
│  3. Fetch Lake Formation LF-Tags    ← PII/sensitivity                      │
│  4. Combine & generate embeddings   ← Merge + Titan (1024-dim)             │
│  5. Load graph into Neptune         ← Create vertices/edges                │
│  6. Create DynamoDB table           ← Query store (synodb_queries)         │
│  7. Load seed queries               ← Initial patterns                     │
│                                                                             │
│  Result: Knowledge graph ready for NL queries                              │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ WHAT GETS CREATED ─────────────────────────────────────────────────────────┐
│                                                                             │
│  NEPTUNE GRAPH:                                                            │
│    • 1 database vertex                                                     │
│    • 3 table vertices (with embeddings)                                    │
│    • 23 column vertices (with embeddings, PII tags)                        │
│    • 8 business term vertices (with embeddings)                            │
│    • 35 edges (contains, has_column, references, described_by)             │
│                                                                             │
│  DYNAMODB (synodb_queries):                                                │
│    • 8 seed query items (with embeddings)                                  │
│    • GSI: workload-index, success_count-index                              │
│                                                                             │
│  EMBEDDINGS:                                                               │
│    • 50 total vectors (1024 dimensions each)                               │
│    • Via AWS Bedrock Titan (amazon.titan-embed-text-v1)                    │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ EXAMPLE QUERIES ───────────────────────────────────────────────────────────┐
│                                                                             │
│  "What is the total portfolio value?"                                      │
│  → SELECT SUM(market_value) FROM positions                                 │
│                                                                             │
│  "Show me revenue by region"                                               │
│  → SELECT region, SUM(revenue) FROM sales GROUP BY region                  │
│                                                                             │
│  "Which customers have the highest orders?"                                │
│  → SELECT c.name, SUM(o.total) FROM orders o                               │
│     JOIN customers c ON o.customer_id = c.customer_id                      │
│     GROUP BY c.name ORDER BY SUM(o.total) DESC                             │
│                                                                             │
│  "What is the monthly trend in AUM?"                                       │
│  → SELECT DATE_TRUNC('month', date), SUM(market_value)                     │
│     FROM positions GROUP BY DATE_TRUNC('month', date)                      │
└─────────────────────────────────────────────────────────────────────────────┘
