## Critical Fix Applied

### Problem
All 3 Silver jobs initially FAILED with:
```
AnalysisException: Cannot modify the value of a static config: spark.sql.extensions
```

### Root Cause
Iceberg configuration was being set via `spark.conf.set()` in the PySpark scripts **after** SparkSession was already initialized by AWS Glue. Static Spark configs like `spark.sql.extensions` must be set at **job startup**, not at runtime.

### Solution Applied
1. **Removed** all `spark.conf.set()` calls from Silver scripts:
   - `silver_funds_clean.py`
   - `silver_market_data_clean.py`
   - `silver_nav_clean.py`

2. **Added** Iceberg configuration to Glue job `DefaultArguments`:
   ```json
   {
     "--datalake-formats": "iceberg",
     "--conf": "spark.sql.extensions=org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions --conf spark.sql.catalog.glue_catalog=org.apache.iceberg.spark.SparkCatalog --conf spark.sql.catalog.glue_catalog.catalog-impl=org.apache.iceberg.aws.glue.GlueCatalog --conf spark.sql.catalog.glue_catalog.warehouse=s3://your-datalake-bucket/ --conf spark.sql.catalog.glue_catalog.io-impl=org.apache.iceberg.aws.s3.S3FileIO"
   }
   ```

3. **Uploaded** fixed scripts to S3

4. **Updated** all 3 Glue jobs with new configuration

5. **Restarted** jobs in parallel at 13:17:37 EST

done

## Key Learnings

1. **Static Spark configs must be set at job startup**, not runtime via `spark.conf.set()`
   - Use Glue job `DefaultArguments` with `--conf` flag
   - Applies to: `spark.sql.extensions`, `spark.sql.catalog.*`

2. **Iceberg requires `--datalake-formats: iceberg`** in Glue job config
   - Without this flag, Iceberg libraries may not load correctly

3. **Always fix Bronze data generation type issues early**
   - PySpark DoubleType requires float literals (9999.0), not ints (9999)
   - Error manifests late in the pipeline

4. **Background monitoring scripts are useful but require proper exit handling**
   - Always check job status before assuming success
   - Use jq to parse JSON outputs
